- Next.js uses file-system routing where folders are used to create nested routes.
- page.tsx is a special Next.js file that exports a React component, and it's required for the route to be accessible.
- A special layout.tsx file to create UI that is shared between multiple pages
- The <Layout /> component receives a children prop. This child can either be a page or another layout. All pages/layouts that in the same folder with layout.tsx are it's child.
=> On navigation, only the page components update while the layout won't re-render. This is called partial rendering which preserves client-side React state in the layout when transitioning between pages.

7.
- Nó liệt kê 3 cách lấy/tương tác data: 
+ Database query: SQL native hoặc ORM. Cái này có thể dùng trong 2 cái dưới.
+ Dùng API layer: 3rd-party service (Nest, Sail,... server).
=> But in Next.js, you can create API endpoints using Route Handlers.
+ React Server Components: By default, Next.js applications use React Server Components. Fetching data with Server Components is a relatively new approach, few benefits: 
.) use async/await syntax without needing useEffect, useState or other data fetching libraries
.) run on the server, so you can keep expensive data fetches and logic on the server, only sending the result to the client
.) query the database directly without an additional API layer
8.
- Static rendering: data fetching and rendering happens on the server at build time (when you deploy) or when revalidating data. Nhưng nó có rất nhiều lợi ích: nhanh tại load sẵn tại server mà, Reduced Server Load (user nào truy cập nó cũng lấy cached data), SEO (data cứng nên search engine dễ search).
=> Static rendering is useful for UI with no data or data that is shared across users, such as a static blog post or a product page. 
- Dynamic rendering: content is rendered on the server for each user at request time (when the user visits the page). Lợi ích nhiều, ex: real time data.
9. 
- Streaming is a data transfer technique that allows you to break down a route into smaller "chunks" and progressively stream them from the server to the client as they become ready.
- Streaming works well with React's component model, as each component can be considered a chunk. Trên UI, cái nào gọi API xong thì nó load lên trước cho user interact trước, siêu đã.
- There are two ways you implement streaming in Next.js:
+ At the page level, with the loading.tsx file (which creates <Suspense> for you).
+ At the component level, with <Suspense> for more granular control.
- Streaming the whole page: loading.tsx is a special Next.js file built on top of React Suspense. It allows you to create fallback UI to show as a replacement while page content loads.
=> Vì file dashboard/loading.tsx có level to hơn 2 page dashboard/invoices/page.tsx và dashboard/customers/page.tsx, nên file loading.tsx sẽ apply cho cả 2 page đó. Nếu ko muốn apply cho 2 page đó, chỉ muốn apply cho /dashboard/page.tsx thôi, thì mình có thể để loading.tsx và page.tsx vào folder /dashboard/(overview)/ (/dashboard/(overview)/loading.tsx và /dashboard/(overview)/page.tsx). Thì cái phần () đó gọi là route group, nó sẽ tự bị xoá trên URL path, dùng để chia application ra nhiều section (e.g. (marketing) routes and (shop) routes) hoặc cho nhiều team deveveloper để dev 1 application lớn.  
- Streaming a component: dùng component React <Suspense>
- Dùng <Suspense> ở đâu là tuỳ mình và tuỳ app của mình, mình có thể nhóm 1 cụm 4 cái <Card> và cho vào <CardWrapper> rồi Streaming cho <CardWrapper>, có thể Streaming cả page như phía trên (dùng loading.tsx file), có thể Streaming từng component nhỏ nhất.

- Client hooks:
+ useSearchParams(): lấy URL query params.
+ usePathname(): lấy URL path.
+ useRouter(): điều hướng route với client component. 
- "use client" - This is a Client Component, which means you can use event listeners and hooks.
- prop "searchParams" là built-in prop cho mọi page.tsx, chức năng nó y chang hook useSearchParams() nhưng nó xài cho server component. useSearchParams() thì xài cho client component.
- 1 cách để apply input debounce là dùng useDebouncedCallback() hook của npm package "use-debounce". 
- Đối với client component, mình ko nên fetch data trỏng, tại this would expose your database secrets (remember, you're not using an API layer). Instead, you can fetch the data on the server, and pass it to that component as a prop.

- React Server Actions là func để chạy đc code async ngay trên server, mình khỏi cần tạo server API làm gì. Func này có thể đc gọi từ Client và Server component.
- React Server Actions include features like encrypted closures, strict input checks, error message hashing, host restrictions, and more.
- Nextjs tự hào với thằng <form> trong server component gọi server action, UI chưa cần load JS xong thì form vẫn hoạt động đc.
- Add 'use server' vào đầu file, thì tất cả các func trong file đó đều là server action, hoặc add 'use server' vào đầu func nào đó bất kì cũng đc, nhưng next khuyến nghị nên để ra 1 file riêng.
- Next.js has a client-side router cache that stores the route segments in the user's browser for a time. Along with prefetching, this cache ensures that users can quickly navigate between routes while reducing the number of requests made to the server. Ví dụ: cache ds invoice ở route (page) invoice, ảo thật, redis tuổi. 
- import { revalidatePath } from 'next/cache'; để clear cache ở 1 route URL nào đó. Ví dụ: vừa thêm data vào ds invoice, thì mình cần clear cache ở page ds invoice. 
=> Ở local ko gọi cái này nó vẫn update data thành mới nhất, nhưng trên production thì ko nhé. Khi sửa 1 record nào đó, mình phải revalidatePath toàn bộ page dùng nó. Rất bất tiện, nên ngta thường xài revalidateTag.
+ revalidatePath: Invalidate specific routes/pages cache.
+ revalidateTag: Invalidate shared data resources cache.
- [something] khi đặt tên folder là Dynamic route segment. Vd: dashboard/invoices/[id]/edit/page.tsx
- prop "params" là built-in prop cho mọi page.tsx, nó dùng để lấy URL params
- Bỏ id vào như này: <form action={updateInvoice(id)}> sẽ ko work, phải dùng JS bind() (This will ensure that any values passed to the Server Action are encoded.).

- Adding Metadata good for SEO. It works behind the scenes, embedded within the page's HTML, usually within the <head> element.
- Các loại common metadata: title, description, keywords (để indexing page của mình), Open Graph (để share trên mạng xã hội), favicon (icon cạnh title).
<title>Page Title</title>
<meta name="description" content="A brief description of the page content." />
<meta name="keywords" content="keyword1, keyword2, keyword3" />
<meta property="og:title" content="Title Here" />
<meta property="og:description" content="Description Here" />
<meta property="og:image" content="image_url_here" />
<link rel="icon" href="path/to/favicon.ico" />
- Có 2 cách add:
+ Config-based: Export a static metadata object or a dynamic generateMetadata function in a layout.js or page.js file (ưu tiên đặt ở layout, nếu layout ko có mới đặt ở page).
.) Object ở page/layout con sẽ có độ ưu tiên lớn hơn thằng cha.
.) Có thể dùng biến %s ở title để ghép các title ở page/layout con vào title template được đặt ở page/layout cha.
+ File-based: Next.js has a range of special files that are specifically used for metadata purposes. For ex:
.) favicon.ico, apple-icon.jpg, and icon.jpg: Utilized for favicons and icons
.) opengraph-image.jpg and twitter-image.jpg: Employed for social media images
.) ...
